# AlgoWire — Static UI Prototype

This directory contains the high-fidelity static frontend prototype for **AlgoWire**, an immersive, developer-focused blog platform. It serves as the design blueprint and interactive mockup for the entire application ecosystem.

## 🎨 Design Vision

- **High-Tech Aesthetic**: Inspired by modern IDEs and high-tech interfaces (WayneTech/Glassmorphism).
- **Responsive Layouts**: Fully optimized for mobile, tablet, and desktop viewing.
- **Interactive Components**: Hover effects, smooth transitions, and tactile UI elements built with vanilla CSS and JavaScript.

## 📁 Subfolder Structure

### Core Pages
- `index.html`: The main portal featuring hero sections and dynamic article grids.
- `article.html` / `post.html`: Detailed reading experience with optimized typography.
- `category.html` & `authors.html`: Discovery pages for filtering content.
- `about.html` & `contact.html`: Brand identity and user engagement pages.

### Admin Interface (`/admin`)
A complete static mockup of the administrative suite:
- `dashboard.html`: Analytics and overview.
- `articles.html`, `categories.html`, `authors.html`: Management lists.
- `article-form.html`, `author-form.html`: Creation and editing interfaces.
- `login.html`: Secure authentication entry point.

### Assets & Styling
- `css/`: Modular CSS architecture for improved maintainability:
  - `base.css`: Global variables, reset, layout, and animations.
  - `navbar.css`, `footer.css`, `components.css`: Reusable UI elements.
  - `cards.css`, `article.css`, `authors.css`, etc.: View-specific styles.
- `js/`: Interactive logic for theme switching, navigation, and UI states.
- `assets/`: Images, icons, and media files.

## How to Use

Simply open any `.html` file in a modern web browser to explore the design. For the best experience, including interactive features like theme switching, it is recommended to use a local development server (e.g., Live Server extension in VS Code).
